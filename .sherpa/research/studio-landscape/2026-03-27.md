# Developer Tooling & Studio Landscape — 2026-03-27

## TL;DR

March 2026 is the inflection point where AI coding tools fully shifted from "copilot" to "agent." MCP is now universal infrastructure (97M monthly SDK downloads, every major provider supports it). The competitive field for Sherpa Studio is sharpening: Cursor is now a full agent platform with cloud execution; OpenAI's Codex went from toy to serious contender with GPT-5.1-Codex-Max; JetBrains is abandoning human pair programming for agentic AI. A new meta-category is emerging: **agent skill management** — tools to organize, index, and share skills across multiple AI coding agents.

---

## 1. AI Coding Tool Landscape

### The Big Four (and emerging players)

| Tool | Latest Status | Key Moves (Q1 2026) |
|------|--------------|---------------------|
| **Cursor** | Full agent platform | CLI launched Jan 2026 with Plan/Ask agent modes, cloud handoff for background tasks, one-click MCP auth, self-hosted cloud agents |
| **Claude Code** | Terminal-first agent | Browser-based IDE at claude.ai/code, native VS Code + JetBrains extensions, background task execution |
| **Windsurf** | AI-first IDE | Supercharges existing IDE, MCP support, competing on integration rather than raw agent capability |
| **OpenAI Codex** | Serious contender | GPT-5.1-Codex-Max default model, IDE extension for VS Code/Cursor/Windsurf, agent skills system with progressive disclosure, CLI + desktop app |
| **GitHub Copilot** | Workspace focus | Copilot Workspace for issue-to-PR flows, still largest install base |

### Emerging: Antigravity
- New AI-native editor rebuilding the editor from scratch with AI as core feature, not add-on

### New: Skills Management Tools
A whole new category has appeared — tools to manage agent skills across platforms:

- **Chops** (macOS, open source) — Browse, edit, manage skills across Claude Code, Cursor, Codex, Windsurf, Amp, Copilot, Aider
- **Agent Skills** (agentskill.sh) — Indexes 100k+ skills across Claude Code, Cursor, Codex, Windsurf
- **Agent-i-Kit (akm)** — CLI for searchable, indexed skill stash; handles cross-platform skill formats
- **Killer-Skills** (npx killer-skills) — Install skills across 19+ IDEs

**Signal for Sherpa:** Skills-as-artifacts are becoming a real ecosystem. There's a management layer opportunity here that nobody owns yet.

### Velocity Data
- Teams report **30–50% velocity improvements** on routine feature development with Tier 1 agentic tools
- Every major tool has **multi-agent orchestration on its Q2 2026 roadmap**
- Claude Sonnet 4.6 introduces **1M token context window** (beta), 128K output, Agent Teams, adaptive thinking with effort controls

## 2. MCP Ecosystem

### Scale
- **97 million monthly SDK downloads** (March 2026), up from ~2M at launch (Nov 2024)
- **Every major AI provider** now ships MCP-compatible tooling: OpenAI, Google, xAI, Mistral, Cohere

### 2026 Roadmap (published ~Mar 13)
Key focus areas from the official MCP roadmap:

1. **Production hardening** — Security, authorization, reliability for real-world use
2. **Extensions ecosystem maturation**
3. **On the horizon:** Triggers and event-driven updates, streamed/reference-based result types, deeper security/auth

### v1.27 Release (Feb 2026)
- Quiet but significant milestone — MCP crossing from experimental to production infrastructure
- Description from practitioners: "Less new magic, more boring integration done consistently — fewer bespoke bridges, clearer ownership of side effects, same tool server runs in IDE and desktop assistant"

### What this means for Sherpa
MCP is now table stakes. Any tool that doesn't support it will be left behind. The differentiation is shifting from "supports MCP" to "what you build on top of MCP." The protocol itself is becoming the plumbing, not the product.

## 3. Notable Product Moves

### Cursor: Full Agent Platform
- **CLI** (Jan 2026): Agent modes (Plan, Ask), cloud handoff for background tasks, one-click MCP auth
- **Cloud Agents**: Run coding agents in the cloud for continuous assistance
- **Self-hosted Cloud Agents**: Code, build outputs, and secrets stay on internal machines — agent executes in your infra
- **This is a platform play**, not just an IDE anymore

### OpenAI Codex: Serious Upgrade
- **GPT-5.1-Codex-Max** now default — "faster, more intelligent, more token-efficient at every stage of development"
- **Agent Skills system** with progressive disclosure: skills are SKILL.md files with metadata, loaded on-demand
- **IDE Extension** works with VS Code forks (Cursor, Windsurf) — not just ChatGPT
- **CLI + Desktop App** — full local agent experience
- OpenAI is clearly investing heavily in developer tooling as a primary surface

### JetBrains: Retiring Human Pair Programming
- **"Central"** — new agentic AI development tool, replacing Code With Me
- Retiring human pair programming feature to "focus efforts" on agentic AI
- Controversial move — some developers devastated by loss of Code With Me
- **Signal:** JetBrains sees the future as human-to-AI collaboration, not human-to-human through the IDE

### Claude Code: Expanding Reach
- Browser-based IDE at claude.ai/code (not just terminal anymore)
- Native extensions for VS Code (+ Cursor, Windsurf) and JetBrains
- Background task execution
- Anthropic positioning as the "everywhere" coding agent

## 4. Industry Narrative

The dominant narrative in March 2026:

> "We've moved from AI as a helpful assistant to AI as a collaborative partner capable of autonomous action."

Key reframes from industry analysis:
- **From copilot → agent**: Tools are expected to take autonomous action, not just suggest
- **From IDE feature → platform**: Cursor, Codex, Claude Code all building full platforms
- **From single-agent → multi-agent orchestration**: Q2 2026 roadmap for every major tool
- **From "supports MCP" → "built on MCP"**: MCP is now infrastructure, not feature

## 5. Implications for Sherpa Studio

### Opportunities
1. **Skills management/registry** — Nobody owns this layer cleanly yet. Open source tools are emerging but fragmented
2. **Multi-agent orchestration tooling** — Every tool is heading here but nobody has it right yet
3. **MCP-native workflow design** — Moving beyond "support MCP" to "build compelling workflows on MCP"
4. **Enterprise self-hosted agents** — Cursor is exploring this; demand is clearly there

### Threats
1. **Cursor becoming a platform** — If Cursor owns the agent execution layer, middleware tools get squeezed
2. **OpenAI's skill system** — Codex's progressive disclosure and SKILL.md format could become the standard
3. **JetBrains Central** — Large installed base pivoting hard to agentic
4. **Commoditization of MCP support** — No longer a differentiator, just expected

### Strategic Positioning
The field is consolidating around a few agent platforms (Cursor, Claude Code, Codex) with MCP as the universal plumbing. Sherpa Studio's sweet spot may be the **orchestration and workflow layer** — the thing that connects multiple agents, manages skills across platforms, and provides the developer experience layer on top of the raw agent infrastructure. The skills management category is a strong early signal of unmet demand.
