## 🧭 Sherpa Studio Positioning Implications

### Where the space is going
1. **Background agents are the next battleground.** Claude Code's `/loop`, Cursor Automations, Codex plugins — everyone is building cron-like persistent agents. Sherpa Studio's "agentic engineering workflow suite" needs to clearly own a layer *above* these tools, or *between* them.

2. **The IDE is commoditizing.** Free tools (Antigravity, Gemini CLI) are competing with Cursor's $20/mo. The value is moving to: orchestration, team workflows, enterprise integration, and who manages the agents — not who builds the UI.

3. **MCP is the integration layer.** If Sherpa Studio ships MCP servers for its workflow concepts, they plug into every major tool. This is leverage.

4. **Claude Code is the CLI darling.** NYT coverage + `/loop` + sub-agents = Anthropic is winning the developer mindshare battle. Sherpa's Claude Code integration story should be front and center.

5. **JetBrains/IntelliJ support is a signal.** The enterprise dev population (Java, Kotlin, Spring) is finally entering the AI IDE market via ACP. Enterprise teams = consulting opportunities for Sherpa.

### Competitive gap to watch
Cursor has Team Marketplaces. Codex has plugin stores. No one has a compelling *consulting-firm-as-workflow-layer* story yet. That's Sherpa's differentiation if it's articulated well.
