## 🧭 Sherpa Studio Positioning Notes

**Where the market is going:**
1. Every major player (JetBrains, GitHub, Cursor, Google) is building **agentic infrastructure** — the tooling layer is commoditizing
2. **Governance + observability** is emerging as the next competitive dimension (JetBrains Central, enterprise Copilot)
3. **Skills/plugins as cross-platform primitives** are becoming standard — Firebase agent skills, VS Code Agent Skills, OpenClaw skills, Codex plugins all speaking the same language
4. The "vibe coding → ship production apps" narrative is mainstream. This legitimizes and expands the Sherpa Studio audience

**Sherpa Studio's differentiation opportunity:**
- **Workflow *design* layer** — not just "run agents" but design, version, and iterate on how agents work together. Nobody owns this yet.
- **The consultant angle** — Sherpa combines the studio (tools) + the practice (expertise). No other tool vendor has this dual-sided value prop.
- **MCP as distribution** — WavePoint's MCP server is a proof point. Sherpa Studio could become the place you build and share MCP-powered workflows.
- **Human-in-the-loop design** — Claude Code's new "defer" permission feature is moving in this direction. Sherpa Studio should be opinionated about *when* agents should pause for human review vs. run autonomously.

**Competitors to watch closely:**
- Augment Code's "Intent" (spec-driven multi-agent) — most similar to Sherpa Studio's positioning
- JetBrains Central (governance layer) — could become table stakes for enterprise
- Google Antigravity (browser-based full-stack agent) — lowest friction entry point, democratizing threat
