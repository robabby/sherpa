# Studio Landscape Research
**Week of March 31, 2026**

Monthly scan of the AI dev tooling ecosystem — key signals for Sherpa Studio positioning.

## 🔑 Top Signal This Week

**The agentic infrastructure layer is now the competition.** The race has moved past "AI coding assistant" into who owns the *orchestration and deployment plane* for autonomous agents. JetBrains Central, Cursor's self-hosted cloud agents, and Claude's Dispatch+computer-use combination all launched within the same week. This isn't coincidence — it's convergence.

## 🔵 GitHub Copilot / VS Code

### Agent Mode GA
- Copilot agent mode is now GA on VS Code and JetBrains IDEs
- Multi-step autonomous agent: determines which files to edit, runs terminal commands, iterates on errors — no manual intervention
- VS Code added `git.addAICoAuthor` config to credit AI co-authorship in commits (chat+agent mode or all AI code)
- **Agent Skills**: reusable instruction sets that teach agents how to perform specific tasks (build pipelines, boilerplate, coding standards); pulled from `.github/skills/` or `~/.github/skills/`

- Copilot Chat available across VS Code, VS, JetBrains, Xcode, Eclipse, GitHub.com, mobile

**Sherpa angle:** GitHub's Agent Skills are conceptually close to what Sherpa Studio skills aim to do. The `~/.github/skills/` pattern is worth studying. Copilot's distribution advantage (bundled everywhere) remains the moat.

## 🟠 JetBrains Central (March 24, 2026)
**This is the biggest structural move of the week.**

- JetBrains announced **Central** — described as "the control and execution plane for agent-driven software development"
- Not an IDE feature — production infrastructure for orchestrating, governing, and running autonomous agents across IDEs, CLIs, pipelines, and collaboration tools

**Sherpa angle:** JetBrains Central is the most direct structural competitor to what a mature Sherpa Studio could be — an orchestration platform for agentic workflows. The framing of "distributed system of agents, environments, and workflows" is *exactly* the Studio vision. They have a massive installed base. Key differentiator for Sherpa: **independence, composability, AI-native from the start**.

## 🎯 Positioning Notes for Sherpa Studio

### What's shifting
1. **The incumbent platforms are building the workflow layer** — JetBrains Central, Cursor Automations, Copilot Agent Skills all launching simultaneously. The window to establish Sherpa Studio as *the* workflow orchestration layer for teams is narrowing.

2. **Computer use changes the game** — Claude's computer use integration means agents can now operate UIs they weren't built for. This eliminates some of the "connector gap" that previously required custom MCP servers. Studio needs a clear answer: are we building on top of computer use, or are we the alternative?

3. **Self-hosted is table stakes** — Cursor GA'd self-hosted cloud agents. Any Studio offering for security-conscious teams needs a clear self-hosted story.

4. **Code With Me dying = interesting signal** — JetBrains retiring human pair programming in favor of agentic collaboration is the starkest statement yet that the industry believes agents > collaboration UX. Sherpa's "human + AI" workflow framing needs to be intentional about where humans stay in the loop.

### Differentiation angles that still hold
- **Consulting-first, tool-second**: Sherpa Studio is the *practice* of agentic engineering, not just tooling
- **AI-native from day one**: Not bolted onto legacy IDE infrastructure
- **Composable and open**: MCP-native, not locked to one provider
- **Honest craftsmanship**: The incumbents are moving fast and breaking things — Sherpa's foundation values are a genuine differentiator with teams that care about reliability
