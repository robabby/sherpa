## Demo Proposal: Behavioral Governance Framework

**Format:** 5-min live demo + 3-min Q&A (AI Tinkerers standard)  
**Track:** Dev Tools  
**Target event:** April 13, 2026

### Proposed Talk Title

> **"Governance-as-Code: How I Kept 472 PRs Coherent With Claude Code and a Behavioral Framework"**

Alt options:
- "The .md's That Made It Work: Behavioral Governance for Production Agentic Dev"
- "How I Shipped 472 PRs Solo in 11 Weeks (And What Almost Broke It)"
- "Beyond Prompt Engineering: Governance Infrastructure for Long-Running Claude Code Sessions"

### 2–3 Sentence Abstract

Over 11 weeks, I shipped 472 PRs across a Next.js web app and 6 native Apple apps — solo — using Claude Code as my primary engineer. The secret wasn't the prompts: it was the behavioral governance infrastructure I built to keep agents coherent across thousands of tool calls, context resets, and multi-day sessions. I'll show the framework live: AGENTS.md conventions, the 91-component workflow suite, behavioral drift signals, and the MCP server that lets agents take real actions with guardrails.

### Technical Depth Statement

I built the governance framework while building the product it governs. It's not theoretical — it's the operational infrastructure behind a production codebase with 472+ PRs merged. I'll show real YAML configs, real .md files, real MCP tooling, and the failure modes I encountered and fixed. No slides; live code only.

### Demo Outline (5 minutes)

**Minute 1: The problem** (live in terminal)
- Show a raw Claude Code session without governance: what drift looks like
- The context reset problem: how agents "forget" behavioral constraints mid-session
- Why prompting alone doesn't scale across 11 weeks of daily sessions

**Minute 2: The solution architecture** (live in Sherpa Studio)
- AGENTS.md as behavioral anchor: what goes in, what it does, why it works
- The workflow component suite: 91 standardized interaction patterns
- Show the component tree live — not a slide, the actual app

**Minute 3: MCP tooling** (live demo)
- MCP server running locally — show tool list
- How agents call tools with behavioral constraints enforced at the protocol level
- One live call: agent executes a workflow, governance triggers, result

**Minute 4: Production evidence**
- Git log: 472+ PRs, 11 weeks, consistent style, zero behavioral catastrophes
- Show one PR diff — generated by Claude Code under the framework
- Brief note on where the framework is heading as Sherpa Studio scales

**Minute 5: Open question for the room**
- "I built this to solve my problem. What's your governance problem?"
- Invite collaboration, not product pitch
- Hand off to Q&A

### Screening Question Answer

"I've spent the past 12 weeks in production agentic development — 472+ PRs across 7 platforms, solo, with Claude Code as my primary engineer. My contribution is a framework and workflow system I built to solve the hardest part of long-running AI dev sessions: behavioral governance. Not prompt engineering — operational infrastructure that keeps agents coherent across context resets, multi-day sessions, and thousands of tool calls. I built the 91-component workflow suite and MCP server that back it, and I'll show live code, real production artifacts, and the failure modes that shaped the design."

## AI Tinkerers Demo Principles (for prep)

From their own published guide:
- **Show, Don't Tell** — live demo, no decks
- **Expose the Internals** — share code, configs, workflow graphs
- **Go Deep** — technical sophistication, complex orchestration
- **Teach, Don't Sell** — insights and decisions, no startup pitch
- **Embrace Imperfection** — half-baked is fine, fragile demos are welcome
- **Push the Edges** — what's novel, what's surprising

Rob's framework clears every bar. The governance angle is specifically novel — most agentic dev talks at this level are about what agents can do, not how to keep them reliably on-track across thousands of calls.
