## Cover Letter

Linear is building the product development system for the era when agents ship alongside engineers. I've spent the past year building the other half of that equation — the agent-side infrastructure that makes that collaboration actually work.

Sherpa Studio is an agentic engineering workflow suite I built solo: multi-agent dispatch across nine backends, behavioral agent definitions, an MCP server for structured context injection, an initiative lifecycle engine, and a knowledge management layer that keeps project context coherent as it evolves. I didn't build it as a demo. I used it to ship WavePoint — a Next.js web app, six native Apple apps, and an MCP server — in eleven weeks across 472+ pull requests.

The reason I'm writing to you specifically about the AI role: I noticed Linear recently expanded its MCP server to support initiatives, project milestones, and updates from tools like Cursor and Claude. The initiative lifecycle engine in Sherpa Studio and Linear's initiatives feature are solving adjacent versions of the same coordination problem — how do you give agents enough structured context to operate with confidence instead of guessing? I've been living in that problem daily, and I have strong opinions about what works and what doesn't.

The thing I've learned is that the hard part isn't the AI integration itself — it's the product design around it. When should an agent act autonomously versus surface a decision? How do you structure context so it degrades gracefully when stale? How do you build workflows that make humans and agents more effective together without making either one a bottleneck? These are product engineering problems, not ML problems, and they're what I find most interesting about what Linear is doing.

Before Sherpa and WavePoint, I spent 13+ years as a frontend and product engineer, most recently at Staff level. I've built component systems, led design system adoption, and owned product surfaces end-to-end. I write React and TypeScript daily and care about craft the way Karri talks about it — not as a slower path, but as the only one that compounds.

I'd welcome Linear's paid work trial format. Building alongside the team is the best way to show what I'd bring, and I'm confident in what it would show.
