# Grammar & Mechanics

Rules for capitalization, numbers, punctuation, and formatting — so every contributor (human and AI) produces consistent copy.

---

## Capitalization

- **Headings:** Sentence case ("Define behavioral constraints," not "Define Behavioral Constraints")
- **Product names:** Title case for proper nouns only — Sherpa, Studio, WavePoint
- **Framework concepts:** Lowercase — "initiative," "behavioral constraint," "quality gate," "open source"
- **Acronyms:** All caps, no periods — AI, CLI, MCP, API

## Numbers

| Rule | Example |
|------|---------|
| Spell out one through nine | "three sessions," "five agents" |
| Numerals for 10+ | "12 engagements," "91 components" |
| Always numerals for percentages | 5%, 95% |
| Always numerals for versions, data, measurements | Node 20, 2,410 questions |
| Commas in numbers over 999 | 1,000 not 1000 |

## Dates

- **Code and frontmatter:** ISO 8601 — `2026-03-14`
- **Prose:** Human-readable — "March 14, 2026" or "March 2026"

## Punctuation

- **Serial comma:** Always — "strategy, governance, and execution"
- **Em dash:** Spaces on both sides — "governance — not just strategy — matters"
- **Periods:** One space after. No periods in headings. Periods in bullet points only for complete sentences.
- **Backticks:** Code references, file paths, CLI commands inline — `defineConfig()`, `docs/agents/roles/`

## Lists

- **Numbered lists** for sequential steps
- **Bullet lists** for unordered items
- Capitalize the first word of each item
- Parallel structure — all items start with the same part of speech

## Technical formatting

- Code references in backticks: `sherpa.config.ts`, `pnpm dev`
- File paths in backticks: `packages/studio-core/`
- CLI commands in fenced code blocks, shell prompt omitted:

```
pnpm build
```
