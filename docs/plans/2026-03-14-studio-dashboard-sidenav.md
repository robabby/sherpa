# Studio Dashboard & Side-Nav Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace the 12-panel vertical scroll home page with a persistent left sidebar for navigation and a three-tier operational dashboard.

**Architecture:** shadcn/ui official Sidebar component wraps the root layout via SidebarProvider. Per-section layouts lose their StudioHeader renders — a single shell header with breadcrumb replaces them. Dashboard restructures from 12 equal panels to Action Required / Active Work / Context tiers.

**Tech Stack:** Next.js 16, shadcn/ui Sidebar, Tailwind v4, Lucide icons, `use-persisted-state` hook for collapse persistence.

**Design docs:**
- `docs/initiatives/studio-dashboard-sidenav/design.md` — full architecture, component tree, file plan
- `docs/initiatives/studio-dashboard-sidenav/prototype.html` — visual reference (open in browser)
- `docs/initiatives/studio-dashboard-sidenav/shape.md` — appetite (3 sessions), rabbit holes, no-gos

2. If shadcn Sidebar needs heavy patching beyond 2 known bugs, evaluate custom sidebar
3. If `studio-state-machine` moves to in-progress before Session 3, pause dashboard

### Task 3: Create the shell header component

**Files:**
- Create: `packages/studio-ui/src/studio-shell-header.tsx`

**Step 1: Create the shell header**

This replaces the per-section `StudioHeader`. It renders once in the root layout shell with the mobile sidebar trigger and breadcrumb.

Create `packages/studio-ui/src/studio-shell-header.tsx`:

```tsx
"use client"

import { usePathname } from "next/navigation"
import { Separator } from "@/components/ui/separator"
import { SidebarTrigger } from "@/components/ui/sidebar"
import { StudioBreadcrumb } from "./studio-breadcrumb"

const SECTION_LABELS: Record<string, string> = {
  process: "Process",
  tasks: "Tasks",
  dispatch: "Dispatch",
  workflow: "Workflow",
  docs: "Docs",
  conventions: "Conventions",
  skills: "Skills",
  playbooks: "Playbooks",
  workforce: "Workforce",
  sessions: "Sessions",
  mcp: "MCP",
  activity: "Activity",
}

export function StudioShellHeader() {
  const pathname = usePathname()
  const segments = pathname.split("/").filter(Boolean)
  const sectionKey = segments[0]
  const sectionLabel = sectionKey ? SECTION_LABELS[sectionKey] : undefined

  const breadcrumbSegments = sectionLabel
    ? [{ label: sectionLabel, href: `/${sectionKey}` }]
    : [{ label: "Dashboard" }]

  // Add sub-page segments if deeper than section root
  if (segments.length > 1 && sectionLabel) {
    breadcrumbSegments.push({ label: segments.slice(1).join("/") })
  }

  return (
    <header className="flex items-center gap-2 border-b border-sidebar-border px-4 py-3">
      <SidebarTrigger className="-ml-1" />
      <Separator orientation="vertical" className="mr-2 h-4" />
      <StudioBreadcrumb segments={breadcrumbSegments} />
    </header>
  )
}
```

4. ❌ Building a configurable widget system for the dashboard
5. ❌ Redesigning StudioHeader beyond minimal breadcrumb + trigger
