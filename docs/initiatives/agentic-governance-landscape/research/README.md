# Agentic Governance Landscape — Research

## Summary

Iteration 1 (2026-03-17) surveyed the agentic governance landscape across five vectors: cloud provider stacks, startup funding/traction, open-source projects, buyer demand signals, and coding agent governance specifically. The core finding is a three-layer governance stack forming in the market (infrastructure → compliance → behavioral), with the behavioral layer — where Sherpa operates — genuinely empty.

## Iterations

1. [Iteration 1](iteration-1.md) — Broad landscape survey: 5 vectors, 20+ vendors catalogued, $1.5B+ in funding mapped, three-layer stack identified

## Open Questions

1. **Convention-to-runtime bridge:** How do behavioral conventions (CLAUDE.md, rules) compose with runtime enforcement (Cedar, OPA, cloud IAM)? What's the export/integration surface?
2. **Developer adoption path:** What triggers dev teams to adopt governance tools? Is the ESLint analogy valid? What does PLG look like for governance conventions?
3. **Cross-agent governance:** How do enterprises govern workflows spanning multiple coding agents? What conventions are agent-agnostic?
4. **AI code provenance standard:** Cursor Agent Trace vs. Git AI vs. Tabnine Provenance vs. AI Provenance Protocol — which wins before EU AI Act Article 50 deadline (Aug 2026)?
5. **CAIO role evolution:** Is the CAIO a platform buyer or a framework enabler? Determines enterprise GTM.
