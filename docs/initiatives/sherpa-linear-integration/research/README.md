# Sherpa × Linear Integration Research

## Summary

Iteration 1 surveyed Linear's platform surface across 4 vectors: API/GraphQL, SDK/automation, agent workflows in production, and data model mapping. The core finding: **Linear is the ideal substrate for Sherpa, not a competitor.** Linear handles state (issues, projects, cycles); Sherpa handles governance (behavioral constraints, conventions, lifecycle management). Zero governance agents exist in Linear's 21-agent marketplace — first-mover opportunity.

## Open Questions

1. **MVP implementation scope** — Thinnest vertical slice for Sherpa as Linear Agent? Webhook → governance check → comment?
2. **Linear free tier validation** — API volume, webhooks, label groups on free plan?
3. **Custom fields vs label groups** — Multi-value scenarios (issue touching both research AND content-generation)?
4. **Integration Directory feasibility** — Does Sherpa Consulting qualify without LLC/EIN?
5. **Offline fallback architecture** — Online-only for Linear, filesystem as fallback path?
