## AAIF (Agentic AI Foundation)

The primary governance body for agentic AI standards:

**Formation:** December 2025, under The Linux Foundation

**Founding Platinum Members:**
- Amazon Web Services
- Anthropic
- Block, Inc.
- Bloomberg L.P.
- Cloudflare, Inc.
- Google LLC
- Microsoft Corporation
- OpenAI Inc

**Three Flagship Projects:**
1. **Model Context Protocol (MCP)** — agent-to-tool communication (Anthropic contribution)
2. **AGENTS.md** — repository convention files (OpenAI contribution)
3. **goose** — open-source local-first agent framework (Block contribution)

**Governance Model:** "Transparent and inclusive" — contributors from all backgrounds shape direction. Neutral, open foundation for public interest.

**Events:**
- MCP Dev Summit North America: April 2-3, 2026, New York
- European events forthcoming

## NIST AI Agent Standards Initiative

US government standards body entering the agent space:

**Announced:** February 17, 2026 by NIST's Center for AI Standards and Innovation (CAISI)

**Three Pillars:**
1. Facilitating industry-led development of agent standards and U.S. leadership in international standards bodies
2. Fostering community-led open source protocol development and maintenance
3. Advancing research in AI agent security and identity

**Key Focus Areas:**
- Identity and authorization: how agents are authenticated, permissions scoped, activity logged
- Security controls and risk management: safeguards against misuse, compromise, privilege escalation
- Sector-specific adoption barriers

**Deadlines:**
- March 9, 2026: RFI on AI Agent Security responses due
- April 2, 2026: AI Agent Identity and Authorization Concept Paper responses due
- April 2026+: Listening sessions on sector-specific barriers

## Agent Governance Frameworks

### Governance-as-a-Service (GaaS) — Academic
- Plug-and-play governance protocol between agentic environment and users
- Decouples behavioral enforcement from agent internals
- Governs through observable outputs without access to weights, prompts, or memory
- Trust Factor mechanism: scores agents on compliance and severity-weighted violations
- Graduated enforcement: coercive, normative, adaptive interventions
- [ArXiv paper](https://arxiv.org/html/2508.18765v2)

### Agentic Trust Framework (ATF) — CSA
- Open specification under Creative Commons
- Continuous monitoring with anomaly detection
- Trust earned through observable, explainable actions over time
- [CSA blog post](https://cloudsecurityalliance.org/blog/2026/02/02/the-agentic-trust-framework-zero-trust-governance-for-ai-agents)

### NIST-Based Security Governance (Microsoft)
- Architecting Trust framework based on NIST standards
- [Microsoft Community Hub](https://techcommunity.microsoft.com/blog/microsoftdefendercloudblog/architecting-trust-a-nist-based-security-governance-framework-for-ai-agents/4490556)

### Superagent Open-Source Framework
- Define agents with specific roles and permissions
- Guardrails restricting API calls, data access, execution paths
- Policies defined declaratively (security teams don't modify agent logic)
- [Help Net Security coverage](https://www.helpnetsecurity.com/2025/12/29/superagent-framework-guardrails-agentic-ai/)

### Policy-as-Code Approaches
- Open Policy Agent with Kubernetes for centralized policy orchestration
- [Agent governance at scale (NexaStack)](https://www.nexastack.ai/blog/agent-governance-at-scale)

### Runtime AI Governance Platforms (2026 Survey)
- [AccuKnox survey of runtime governance platforms](https://accuknox.com/blog/runtime-ai-governance-security-platforms-llm-systems-2026)

## llms.txt Standard

Machine-readable website metadata for agents:

- Proposed by Jeremy Howard (September 2024)
- Plain-text markdown file in website root directory
- 600+ websites adopted as of July 2025 (including Perplexity, Anthropic, Cursor, Stripe, Cloudflare, Zapier)
- Not yet an official IETF or W3C standard (community-driven)
- Reduces computational cost for AI to read sites, improves citation accuracy

## Implications for Sherpa

1. **Everything is converging under the Linux Foundation.** AAIF, A2A, AGNTCY, agentgateway — all Linux Foundation projects. Sherpa's governance conventions could be contributed to or aligned with this ecosystem.

2. **NIST is creating regulatory pull for agent governance.** The AI Agent Standards Initiative specifically calls out identity, authorization, security controls, and auditing. Sherpa's initiative lifecycle, behavioral constraints, and activity logs provide exactly these governance primitives.

3. **The governance layer of the stack is still vacant.** The four-protocol stack (MCP + A2A + WebMCP + AG-UI) handles communication. Kubernetes handles runtime. AAIF handles standards governance. But **behavioral governance of agent conduct** — what Sherpa does — has no standard, no project, no working group. This is the gap.

4. **GaaS (Governance-as-a-Service) is the academic validation.** The arxiv paper describes exactly Sherpa's approach: "governs through observable outputs without access to weights, prompts, or memory" and "decouples behavioral enforcement from agent internals." Sherpa should cite this work.

5. **The AAIF MCP Dev Summit (April 2-3, 2026, NYC) is a concrete opportunity.** If Sherpa's convention governance model is ready for external presentation, this is the venue.

6. **`sherpa sync` should generate llms.txt alongside AGENTS.md.** Both are machine-readable metadata about a project. Syncing both from a single source increases Sherpa's surface area in the ecosystem.
