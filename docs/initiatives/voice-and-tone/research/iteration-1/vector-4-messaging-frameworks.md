# Vector 4: StoryBrand & Messaging Frameworks for Technical Services

**Question:** How does StoryBrand adapt for developer-facing consulting? What alternatives work for dual-audience companies?
**Agent dispatched:** 2026-03-13

## Findings

### StoryBrand Limitations for B2B Technical

- **Works for website clarity, breaks down for complex B2B sales.** Single-hero assumption fails for multi-buyer committees (CTO, CFO, procurement have different concerns).
- **Cookie-cutter risk.** If every company follows SB7, messaging sounds identical. Insufficient for differentiation in competitive markets.
- **Effective for static copy, not ongoing content.** LinkedIn, newsletters, podcasts need fresh engagement; StoryBrand's rigid format feels repetitive.
- **No strong examples** of developer tool companies using pure StoryBrand. Implementations cluster around services firms.
- **StoryBrand 2.0** (Jan 2025) updated "empathy and authority" to "empathy and competency" but core SB7 unchanged.

### Alternative Frameworks

**April Dunford's "Obviously Awesome" (Positioning):**
- Five components: Competitive Alternatives, Unique Attributes, Value, Best-Fit Customers, Market Category.
- Starts from "what would customers do if we didn't exist?" Grounds positioning in buyer reality.
- Worked with 300+ B2B tech companies (Google, IBM, Postman). Purpose-built for B2B tech.
- Userlist case study: repositioned from "email automation" to "customer messaging" with immediate conversion lift.

**Andy Raskin's Strategic Narrative:**
- Five elements: Name the Shift, Name the Stakes, Define the Promised Land (NOT your product), Present Magic Gifts, Provide Evidence.
- "Old Game vs. New Game" — companies evangelize a new discipline. Best B2B SaaS companies feel like they're evangelizing a movement.
- Zuora canonical example: "Subscription Economy" instead of "billing software." Uberflip met revenue targets for first time after adopting.
- **Breaks the "arrogant doctor" pattern** — leading with the shift works because everyone in the buying group can agree on it.

**JTBD:**
- Shifts focus from features to underlying tasks. Four Forces model (push, pull, anxiety, habit) provides direct copy template.
- Excels when creating a new category or challenging established approaches.
- Complementary to personas — personas describe who, JTBD describes why.

**Category Design:**
- **90% failure rate** for companies that don't have resources/commitment. 90% of tech IPOs positioned in existing markets.
- Requires organizational alignment beyond marketing. "A death sentence" for most B2B SaaS.
- Not recommended for Sherpa at current stage.

**Peep Laja's B2B Message Layers (Wynter):**
- Four layers: Clarity → Relevance → Value → Differentiation. If they don't get Clarity, nothing else matters.
- Based on thousands of message tests. Diagnostic, not generative.

### Framework Comparison

| Framework | Best For | Weakness |
|-----------|----------|----------|
| StoryBrand | Homepage clarity, simple narrative | Single-hero fails multi-buyer; cookie-cutter |
| Obviously Awesome | B2B tech positioning, differentiation | Doesn't produce copy — produces strategy |
| Strategic Narrative | Sales decks, exec alignment, movements | Requires genuine shift — forced shifts feel hollow |
| JTBD | Buyer motivation, new category education | Abstract without narrative pairing |
| Category Design | Genuine new-category at scale | 90% fail rate |
| Message Layers | Testing/validating messaging | Diagnostic only |

### The Integration Insight

High-performing teams combine: JTBD for customer insight, Dunford for positioning, StoryBrand for homepage narrative, Raskin for sales decks, Message Layers for validation.

### Real Technical Company Examples

- **Stripe**: Documentation IS marketing. Working code on homepage. "Product advocates" instead of BDRs.
- **Twilio**: "Ask Your Developer" billboard — dual-audience in one phrase. Developer community + executive education simultaneously.
- **HashiCorp/Terraform**: Separate pages for developers ("self-service") vs. platform engineers ("the new standard," "policy as code").

### Recommendation for Sherpa: Three-Layer Stack

**Layer 1: Positioning (Dunford)** — Already partially done in product-positioning.md. Complete formally: competitive alternatives, unique attributes, value, best-fit customers, market category.

**Layer 2: Strategic Narrative (Raskin)** — The missing piece. Sherpa has a genuine shift to name: AI-as-tool → agentic-AI-as-workforce, and the governance gap this creates.
1. The Shift: "AI agents are becoming autonomous workers, not just tools"
2. The Stakes: "Organizations that don't govern agentic workflows will ship unreliable AI"
3. The Promised Land: "Teams shipping AI workflows with behavioral governance built in"
4. The Magic Gifts: The framework + the consulting
5. The Evidence: Real engagement data, self-governance

**Layer 3: Per-Persona Messaging (JTBD-informed StoryBrand)** — StoryBrand arc for per-persona website copy, JTBD defines each persona's "job."

### Why NOT Pure StoryBrand

1. Three distinct personas with different problems
2. Dual product (framework + consulting)
3. Anti-hype voice diverges from StoryBrand's emotional urgency
4. Differentiation is strongest card — StoryBrand would flatten it
5. Website proposal's SB7 works as homepage wireframe but shouldn't be elevated to company messaging framework

## Implications

StoryBrand is useful as a homepage wireframe but should NOT be Sherpa's messaging framework. The three-layer hybrid (Dunford positioning + Raskin narrative + JTBD per-persona) is better suited to dual-audience, anti-hype, differentiation-strong companies. Sherpa has a genuine "shift in the world" to name: the governance gap in agentic AI.
